POST /api/v1/user/activate

Second and final step of user registration. Activate the previously created inactive account. This API is meant to be hit when a user clicks a link in their email.

Arguments 

{
	"token": string (required)
}

Response 

{ "status": "success"|"error", "msg": string }

Status codes

  • 200 - success
  • 400 - account already activated
  • 401 - token is invalid
  • 403 - CSRF token validation failed
