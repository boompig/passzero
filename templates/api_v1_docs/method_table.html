Method & path Description Requires login Requires CSRF token
GET /api/v1/csrf_token Get CSRF token for current user's session.
GET /api/v1/entries Retrieve all decrypted entries for the logged-in user
DELETE /api/v1/entries/<int:entry_id> Delete the entry with the given ID.
UPDATE /api/v1/entries/<int:entry_id> Update the specified entry.
POST /api/v1/entries/new Create a new entry for the logged-in user.
POST /api/v1/entries/nuclear Delete all entries for the logged-in user.
POST /api/v1/login Login. On success, update session cookie.
POST /api/v1/logout Logout. Destroy current session.
POST /api/v1/user/activate Second and final step of user registration. Activate the previously created inactive account. This API is meant to be hit when a user clicks a link in their email.
UPDATE /api/v1/user/password Change the master password for the logged-in user.
POST /api/v1/user/recover First step of account recovery for the specified user. Send an account recovery token to the user's email address.
POST /api/v1/user/recover/confirm Second and final step of account recovery for the specified user. This API is meant to be hit when the user clicks the link in a recovery email. Check the token is valid, then nuke all the entries and reset the password.
POST /api/v1/user/signup First step of user registration. Create an inactive user and send an email to the specified email address. The account cannot be logged-into until it has been activated via the activation link in the email.
