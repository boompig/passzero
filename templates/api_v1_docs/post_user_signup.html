POST /api/v1/user/signup

First step of user registration. Create an inactive user and send an email to the specified email address. The account cannot be logged-into until it has been activated via the activation link in the email.

Arguments


{
    "email": string (required),
    "password": string (required),
    "confirm_password": string (required)
}

Response

Success: 

{
    "status": "success",
    "msg": string
}
Error: 
 
{
    "status": "error",
    "msg": string
}

Status codes

  • 200 - success
  • 400 - account already exists
  • 403 - CSRF token validation failed
  • 500 - failed to send email
